Our Solution

A scalable recommendation system that:

  • Parallelizes similarity computations.
  • Uses cosine similarity for matching items.
  • Applies CountVectorizer and TF-IDF for feature extraction.
  • Speeds up processing with parallel similarity computation

System Architecture & Methodology

Data Collection & Preprocessing

Using MovieLens, Books, and Electronics datasets, we construct a sparse user-item matrix and clean metadata for content-based analysis. Missing and duplicate entries are handled for accuracy.

Parallel Similarity Computation

Cosine similarity calculations are parallelized using Python’s ThreadPoolExecutor, speeding up processing of large datasets like reviews and book metadata.

Content-Based Filtering

Movies use CountVectorizer, while books and products use TF-IDF to convert text into feature vectors. Cosine similarity compares these to find top recommendations.

Recommendation Generation

For a given query, the system retrieves similar items the user hasn’t interacted with. These are ranked by similarity scores and returned as personalized recommendations.

Technology Stack

Technologies and tools powering our movie recommendation system

Frontend

Developed using HTML for structure, CSS for styling, and JavaScript for dynamic interactions.

Backend: Flask

Flask is used as the backend framework for handling routes, user inputs, and delivering movie recommendations.

Feature Extraction: TF-IDF

TF-IDF (Term Frequency-Inverse Document Frequency) is used to extract important keywords from product reviews and book metadata for accurate recommendations.

Feature Extraction: CountVectorizer

We use CountVectorizer from scikit-learn to convert movie titles and genres into numerical vectors for similarity analysis.

Similarity Metric: Cosine

Cosine Similarity compares vector angles to find the most similar movies based on extracted features.

Parallel Processing

ThreadPoolExecutor from Python's concurrent.futures speeds up JSON data loading and parsing through multithreading.
